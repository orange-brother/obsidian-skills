# AI Writing Tropes to Avoid

This is the reference list for the human-writer skill. These are patterns that consistently appear in AI-generated text and mark it as non-human. Avoid all of them.

## Words and Phrases

These words are not wrong in themselves, but they appear so often in AI writing that they've become signals. If you notice yourself reaching for one, find a more direct or specific alternative.

### Verbs
- delve (into)
- leverage
- harness
- elevate
- unlock
- navigate
- foster
- facilitate
- underscore
- exemplify
- embody
- streamline
- optimize (when used abstractly)
- revolutionize
- transform (overused in abstract contexts)
- empower
- prioritize (when used loosely)

### Adjectives
- robust
- seamless / seamlessly
- invaluable
- crucial (use "important" or be specific about why)
- vital (same)
- comprehensive (usually meaningless)
- groundbreaking
- game-changing / game-changer
- meticulous / meticulously
- vibrant
- dynamic (as an empty adjective)
- innovative (almost always filler)
- cutting-edge
- state-of-the-art
- holistic
- nuanced (when not actually describing nuance)
- multifaceted
- pivotal
- paramount

### Nouns
- tapestry (especially "rich tapestry")
- landscape (when used metaphorically: "the AI landscape")
- ecosystem (overused for anything interconnected)
- journey (when describing a process or experience)
- realm
- beacon (especially "a beacon of...")
- testament (especially "a testament to...")
- synergy

### Phrases and Openers
- "In today's world" or "In today's fast-paced world"
- "It's worth noting that"
- "It's important to note that"
- "It should be noted that"
- "At the end of the day"
- "Here's the thing" / "Here's the kicker"
- "The bottom line is"
- "Make no mistake"
- "When it comes to"
- "In the realm of"
- "In the landscape of"
- "Dive into" / "Let's dive into"
- "Buckle up"
- "Without further ado"
- "I'm excited to share"
- "In conclusion" followed by a full summary
- "To summarize" when the piece just made the point

## Structural Patterns

### False Profundity
The "It's not X, it's Y" construction. It sounds like insight but usually just reframes the same idea with extra drama.

Bad: "It's not about the code. It's about the people."
Better: "The hardest part of building software is usually the team dynamics, not the technical problems."

### Self-Answered Questions
Starting a sentence with a rhetorical question and immediately answering it creates artificial tension.

Bad: "What's the result? Chaos."
Bad: "Why does this matter? Because everything depends on it."
Better: Just say the thing directly.

### Rule-of-Three Padding
Three parallel items strung together for rhythm rather than because there are actually three important things.

Bad: "It's faster, smarter, and more efficient than anything else on the market."
Better: Pick the one that actually matters and say why.

### Anaphora for Effect
Starting multiple consecutive sentences with the same word to create dramatic rhythm.

Bad: "We built it for you. We built it to last. We built it because we care."
This sounds like an ad from 2009.

### Dramatic One-Sentence Paragraphs
Using a single short sentence as its own paragraph to create emphasis. Used once in a long piece, fine. Used repeatedly, it's a crutch.

Bad:
"The project was almost complete.
Then everything changed.
In an instant."

### Nested Summaries
Restating the same point at multiple levels: within a paragraph, at the end of a section, and again at the conclusion. Say it once.

### Bold-First Bullets
Starting every bullet point with a bold keyword followed by a colon and explanation. This is fine in documentation but it's a giveaway in any text meant to be read as prose.

Bad:
- **Speed**: The system processes requests in under 100ms.
- **Reliability**: Uptime is consistently above 99.9%.

Better: Write it as a sentence or two.

### Excessive Parentheticals
Using parentheses to add qualifications or asides more than once or twice per piece. It creates a stuttering, hedging quality.

### Numbered Lists When Prose Would Flow Better
If the items have a natural relationship or sequence that can be expressed in prose, write prose. Reserve numbered lists for genuine step-by-step instructions.

## Formatting Signals (Recap)

These are covered in the main SKILL.md but worth listing here as a quick reference:

- Em dashes used for dramatic pause or parenthetical (aim for zero per piece)
- Headers in what should be flowing text
- Bold text for emphasis within paragraphs
- Bullet points for content that isn't actually a list
- Nested bullet points in non-technical writing
- Markdown formatting in any context where plain text is expected
